# file-translator

這是一個專門用於檔案內容翻譯的技能，核心目標是「精準、完整、自動化」。

## 實作流程 (Workflow)

當收到翻譯指令時，請嚴格執行以下步驟：

1. **讀取原始檔案:** 使用 `read_file` 讀取使用者指定的檔案內容。
2. **精準翻譯:**
    - 將內容翻譯為**繁體中文**。
    - 保持「一字不漏」且語法精準、自然。
    - **嚴格禁止**修改原始檔案內容。
    - 程式碼區塊中的註解也需視情況翻譯（若使用者有要求），但程式碼本身保持不變。
3. **儲存新檔案:**
    - 輸出路徑為：`{原路徑}/{原檔名}_Cht.{副檔名}`。
    - 如果該檔案已存在，請直接**覆蓋 (Overwrite)**。
    - 使用 `write_file` 寫入內容。
4. **自動 Commit:**
    - 檔案寫入成功後，請直接進行 git commit。
    - **Commit Message 規範:** 必須參考 `git-commit-helper` 技能中的 Conventional Commits 格式。
    - **權限:** 使用者已授權，翻譯完成後**無需再次詢問**，請直接執行 commit。

## 注意事項 (Guidelines)
- 確保翻譯後的 Markdown 格式與原檔完全一致（標題、清單、表格、連結等）。
- 技術專有名詞若無適當對應，請保留原英文或在括號內標註。
- 自動 commit 後，請簡單告知使用者 commit 已完成即可。
