# implementation-completion

專門用於執行結構化計畫中開發任務的技能。專注於可靠的實作、驗證以及維護計畫進度。

## 實作流程 (Workflow)

1. **任務識別**: 僅執行使用者明確要求的特定 Task ID（例如：T1, T2）。
2. **影響評估**:
    *   檢查要求的任務將會異動多少個檔案。
    *   如果受影響的檔案總數超過 **9 個**，你必須向使用者再次確認後方可繼續執行，並說明這是為了避免單次變動過大導致審查困難。
3. **實作與驗證**:
    *   依照任務中「實作細節」的描述套用程式碼變更。
    *   **穩定性**: 確保系統處於可建置狀態。完成任務後**不得出現 Build Error**。
    *   **驗證**: 執行相關的單元測試或指定的驗證步驟。
4. **自動 Commit**:
    *   實作與驗證成功後，立即進行 git commit。
    *   **Commit Message**: 遵循 `git-commit-helper` 技能中的 Conventional Commits 格式。
    *   **授權**: 使用者已預先授權，實作完成後**無需再次詢問**即可執行 commit。
5. **狀態回寫**:
    *   將計畫文件中 **Task 進度表** 的「狀態」欄位更新為 `Review`。
    *   **嚴格限制**: 除非獲得使用者同意，否則你僅被允許修改「狀態」欄位。禁止擅自更動計畫書的其他內容。
6. **回報**: 告知使用者任務已完成並已 commit。

## 注意事項 (Guidelines)
- **繁體中文**: 使用繁體中文與使用者溝通。
- **精準性**: 嚴格遵守任務中提供的「實作細節」。
- **失敗處理**: 若因技術限制或邏輯矛盾導致任務無法完成，請立即停止並向使用者回報問題。請勿嘗試通靈或繞過錯誤。
