# 20250730_關聯商品

## 想法

前台功能只有在商品明細能看到  

沒有審核功能  

PM 希望有修改記錄，留下商品何時被異動

後台功能預計都放 MarketingOperate 服務  
- MarketingOperate，雖然服務名稱與需支功能不太符合，不過 Table 只有 4 個，api 也很少，跟其它服務比起來相對小  
- Spu，服務名稱與需求功能最符合，但是 Table 已達 63 個  
- Product，服務名稱與需求功能稍微符合，但還是不像商品資料，Table 數量也有 37 個  


估計頂多 50 個活動，最多有 50 * 50 = 2500 個商品  

需要有排程定時從 DB 更新至 Redis  
後台異動後要立即更新至 Redis

### 後端 - Spu - 新增 api - 前台取得關聯商品列表  

recomand-associations

被觸發才去更新快取-L1，存在時間 1 小時  
優先抓 快取-L1，若 快取-L1 不存在 或修改時間早於 快取-L2，則再從 快取-L1 算出 快取-L2，再更新到 Redis  
從 快取-L2 取得商品列表，再去 Elasticsearch 查回完整商品資料
