# Gemini CLI 專案指令

## 語言與溝通
- **主要語言：** 與使用者溝通、討論需求、產出系統文件時，一律使用**繁體中文**。
- **例外：** 撰寫 skill 檔案（例如 SKILL.md）與 instruction 檔案（例如 GEMINI.md、CLAUDE.md、copilot-instructions.md）時，一律使用**英文**。

## 回應風格
- 簡潔且技術性。
- 建議變更時，先解釋「為什麼」，再說明「怎麼做」。

## 研究與調查
- **強制分析：** 對於任何複雜需求、架構變更或錯誤調查，在提出計畫前，**必須**使用 `codebase_investigator` 子代理映射依賴關係並驗證假設。
- **上下文效率：** 使用子代理（`codebase_investigator`、`generalist`）「壓縮」大規模研究或批次作業，以保持主要會話歷史精簡，降低 token 消耗。
- **實證驗證：** 在嘗試修復前，優先透過測試或腳本重現已回報的問題，確認失敗狀態。

## 程式碼變更規則

### 何時需要提計畫

**只有程式碼變更**（修改原始碼檔案）才需要提計畫。文件、instruction 檔案與 skill 檔案**不需要**提計畫，可直接進行。

**以下情況可跳過計畫：**
- 使用者明確表示不需要計畫，或
- 使用者表示依照已存在的計畫執行。

### 計畫內容

每個計畫項目必須包含：
- **現狀：** 目前的狀態
- **目標：** 想要達成的目的
- **方法：** 如何實作
- **步驟：** 具體的實作步驟

**計畫位置：**
- 若使用者未指定計畫路徑，執行前須**確認**。

### 執行流程

1. 撰寫／更新計畫，**等待使用者認可**
2. 若未通過，反覆修正計畫直到通過
3. 認可後，依計畫逐步實作
4. 完成所有步驟後，**commit**
